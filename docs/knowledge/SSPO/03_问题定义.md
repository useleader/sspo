# 03_问题定义

## 3.1 从传统偏好优化到半监督设定

### 传统偏好优化的数学形式

在传统设定中，我们假设有**全量标注的偏好数据** $D_L = \{(x^{(i)}, y_w^{(i)}, y_l^{(i)})\}_{i=1}^{n_L}$。

**优化目标**（以DPO为例）：

$$\min_\theta \mathcal{L}_{DPO}(\theta) = -\mathbb{E}_{(x, y_w, y_l) \sim D_L}\left[\log \sigma\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right]$$

**问题**：这需要**100%的偏好标注数据**。

### 为什么这在实际中很成问题？

```
现实情况：

研究人员A: "我想做医学问答模型的偏好优化"
研究人员B: "你们有多少标注数据？"
研究人员A: "只有500条..."
研究人员B: "那不够，至少需要5000-10000条才能训练出好模型"
研究人员A: "但是收集更多数据需要\$50,000和6个月时间..."
研究人员B: "抱歉，这确实是个问题"

与此同时：
研究人员A的团队有大量无标签的医学问答数据（来自医院内部系统）
这些数据包含丰富的医学知识，但没有被利用...
```

### 半监督设定的引入

SSPO采用不同的设定：

**给定数据**：
- $D_L$（少量）：$\{(x, y_w, y_l)\}$ — 有偏好标签的成对数据
- $D_U$（大量）：$\{(x, y)\}$ — **只有prompt和单条response，无偏好标签**

**核心问题**：
> 如何利用 $D_L$ 和 $D_U$ 同时训练，使得性能接近使用全量 $D_L$？

## 3.2 符号系统详解

### 基础符号

| 符号 | 定义 | 示例 |
|------|------|------|
| $x$ | 输入prompt | "如何治疗头痛？" |
| $y_w$ | winning response（被偏好） | "建议服用布洛芬..." |
| $y_l$ | losing response（不被偏好） | "吃点头痛药就行" |
| $\pi_\theta(y\|x)$ | 策略模型（语言模型）的条件概率 | $P_\theta(y\|x)$ |
| $\pi_{ref}(y\|x)$ | 参考模型（通常SFT后的模型） | 用于归一化 |
| $r_\theta(x, y)$ | Reward函数，衡量response质量 | $r_\theta(x,y) = \frac{\beta}{|y|}\log\pi_\theta(y\|x)$ |

### 数据集符号

| 符号 | 定义 | 典型规模 |
|------|------|----------|
| $D_L = \{(x, y_w, y_l)\}$ | Paired data（成对偏好数据） | $n_L \approx 600$ (1%) |
| $D_U = \{(x, y)\}$ | Unpaired data（无标签SFT数据） | $n_U \approx 20,000$ |
| $n_L = \|D_L\|$ | Paired样本数量 | 数百到数千 |
| $n_U = \|D_U\|$ | Unpaired样本数量 | 数万到数十万 |

**关键关系**：$n_L \ll n_U$（通常 $n_U / n_L \approx 30-100$）

### SSPO专用符号

| 符号 | 定义 | 说明 |
|------|------|------|
| $\delta$ | Reward threshold（奖励阈值） | 分界线，高于它视为winning |
| $\delta^*$ | 最优理论阈值 | Theorem 1保证存在 |
| $\hat{\delta}$ | 估计的实际阈值 | 通过KDE从$D_L$学习 |
| $\tilde{s} \in \{0,1\}$ | 伪标签 | 0=losing, 1=winning |
| $\gamma'$ | 自适应系数 | 控制paired/unpaired权重 |
| $\gamma_0$ | 初始系数 | 默认=1 |
| $\gamma_{min}$ | 最小系数 | $= n_L/(n_L+n_U)$ |
| $\lambda$ | 衰减率 | 控制$\gamma'$衰减速度 |

## 3.3 问题形式化详解

### 传统设定（Full Supervision）

**假设**：我们有充足的paired data $D_L$。

**目标**：找到最优策略 $\pi^*$ 最大化偏好概率：

$$\pi^* = \arg\max_\pi \mathbb{E}_{(x, y_w, y_l) \sim D_L}\left[\log P(y_w \succ y_l | x; \pi)\right]$$

使用DPO，这变成：

$$\min_\theta -\mathbb{E}_{D_L}\left[\log \sigma\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right]$$

### SSPO设定（Semi-Supervision）

**假设**：
- 只有少量 $D_L$（如1-10%的数据）
- 有大量无标签的 $D_U$

**目标**：利用两者同时训练，性能接近使用全量 $D_L$。

**形式化**：

$$\min_\theta \mathcal{L}_{SSPO}(\theta) = \gamma' \cdot R_{D_L}(f_\theta) + (1-\gamma') \cdot R_{D_U}(f_\theta)$$

其中：
- $R_{D_L}$：来自paired data的监督损失
- $R_{D_U}$：来自pseudo-labeled unpaired data的损失
- $\gamma'$：自适应权重

## 3.4 核心机制：Reward Threshold伪标签

### 3.4.1 为什么需要"Imaginary Pair"？

Paired data $D_L$ 包含三元组 $(x, y_w, y_l)$，可以直接计算偏好。

但 Unpaired data $D_U$ 只有 $(x, y)$，没有比较对象！

**解决思路**：引入一个**假设的比较对象** $y_b$（hypothetical baseline response）。

想象 $y_b$ 的reward恰好在决策边界上，即 $r(x, y_b) = \delta$。

这样，无标注数据就变成了"虚拟"的比较：

$$\tilde{s} = \begin{cases} 1 & \text{if } r(x, y) > \delta \text{ (winning)} \\ 0 & \text{if } r(x, y) \leq \delta \text{ (losing)} \end{cases}$$

### 3.4.2 伪标签生成规则

对于每个unpaired sample $(x_u, y_u) \in D_U$：

$$\tilde{s} = \mathbb{I}\{r_\theta(x_u, y_u) > \hat{\delta}\}$$

- 如果模型的reward评估认为这个response"够好"（高于阈值），标记为 $\tilde{s}=1$（pseudo-winning）
- 否则标记为 $\tilde{s}=0$（pseudo-losing）

### 3.4.3 伪标签损失函数

有了伪标签后，可以定义与paired data类似的损失：

$$R_{D_U}(f_\theta) = \frac{1}{n_U}\sum_{k=1}^{n_U}\ell(f_\theta, \tilde{s}_k) \cdot \mathbb{P}_{D_U}(s=\tilde{s}_k)$$

其中：
- $\ell$ 是二元交叉熵损失
- $\mathbb{P}_{D_U}(s=1)$ 是unpaired data中winning responses的**先验比例**

## 3.5 自适应调度详解

### 3.5.1 直观理解

为什么需要自适应调度？

```
训练初期（Step 0-100）:
┌─────────────────────────────────────────────────────────────┐
│  Reward分布: winning和losing重叠严重                         │
│                                                             │
│     winning                    losing                       │
│        ╱╲                        ╱╲                          │
│       ╱  ╲                      ╱  ╲                        │
│      ╱    ╲                    ╱    ╲                       │
│  ────┴─────┴────────────────────┴─────┴─────→ reward        │
│              ↑                                              │
│           当前阈值                                           │
│                                                              │
│  问题: 如果现在就大量使用伪标签，会错误传播                    │
│  解决: γ' ≈ 1，主要学习paired data                          │
└─────────────────────────────────────────────────────────────┘

训练后期（Step 500+）:
┌─────────────────────────────────────────────────────────────┐
│  Reward分布: winning和losing分离明显                         │
│                                                             │
│     winning                              losing             │
│        ╱╲                                  ╱╲                 │
│       ╱  ╲                                ╱  ╲               │
│      ╱    ╲                              ╱    ╲              │
│  ────┴─────┴────────────────────────────┴─────┴─────→ reward │
│                            ↑                                │
│                         稳定阈值                             │
│                                                              │
│  优势: 现在可以放心利用大量unpaired data                      │
│  解决: γ' 衰减，开始利用pseudo-labeled data                  │
└─────────────────────────────────────────────────────────────┘
```

### 3.5.2 数学形式

$$\gamma' = \max\left\{\gamma_{min}, \gamma_0 \cdot \exp(-\lambda\tau)\right\}$$

其中：
- $\gamma_0 = 1$：初始时刻，完全依赖paired data
- $\gamma_{min} = n_L/(n_L+n_U)$：最小权重，proportional to数据量
- $\lambda > 0$：衰减率，控制衰减速度
- $\tau$：训练步数/epoch

### 3.5.3 参数选择直觉

| 参数 | 太小 | 适中 | 太大 |
|------|------|------|------|
| $\gamma_{min}$ | 过快引入unpaired，可能overfit伪标签噪声 | 平衡数据利用和稳定性 | 训练后期仍主要依赖paired |
| $\lambda$ | γ'衰减太快，早期就开始用伪标签 | 早期稳定，后期利用unpaired | γ'衰减太慢，利用不足 |

## 3.6 与其他半监督方法的关键区别

### 3.6.1 SSRM vs SSPO

| 方面 | SSRM | SSPO |
|------|------|------|
| 伪标签应用阶段 | 用于训练reward model | **直接用于policy training** |
| 训练流程 | 两阶段：RM训练→策略优化 | 联合训练 |
| 阈值来源 | 启发式或固定 | **KDE从$D_L$学习** |
| 理论基础 | Self-training理论 | **Bayes最优分类+Theorem 1** |

### 3.6.2 SPA vs SSPO

| 方面 | SPA | SSPO |
|------|-----|------|
| 伪标签来源 | 迭代生成（用当前策略） | reward threshold |
| 计算成本 | 高（每次迭代都要重新生成） | 低（一次性KDE估计） |
| 理论基础 | 无明确理论 | **Theorem 1存在性保证** |
| 阈值稳定性 | 可能随迭代波动 | **EMA stabilization** |

### 3.6.3 为什么SSP0的方法更好？

1. **理论基础**：有Theorem 1保证，不是启发式
2. **计算效率**：KDE一次性估计，不需要迭代
3. **阈值稳定**：使用EMA避免剧烈波动
4. **训练稳定**：自适应调度避免confirmation bias

## 3.7 关键约束与假设

### 3.7.1 分布假设

1. **Reward分离假设**：winning和losing responses的reward分布应该有一定分离（但可以有重叠）
2. **分布匹配假设**：$D_U$和$D_L$的数据分布应该相似（同一领域）
3. **一致性假设**：伪标签的质量应该随训练改进

### 3.7.2 理论保证的条件

Theorem 1的成立条件：
- Reward samples是sub-Gaussian分布
- 有足够的paired data ($n_L$足够大)
- winning和losing的均值差 $\mu_w > \mu_l$
